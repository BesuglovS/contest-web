# Контест — система обучения программированию на Python

Веб-платформа для проведения контестов по программированию на языке Python. Позволяет создавать задачи с автоматическими тестами, формировать контесты, управлять группами участников и проверять код в изолированной среде (песочнице) с проверкой стиля PEP8.

## Возможности

- **Авторизация и управление пользователями** — регистрация, вход, разделение на администраторов и участников
- **Задачи (tasks)** — создание задач с условием, форматом ввода/вывода, тестами (публичными и скрытыми)
- **Группы задач (task groups)** — логическая группировка задач
- **Контесты (contests)** — привязка задач к контесту, назначение времени начала/окончания, управление доступом (по пользователям или группам пользователей)
- **Группы пользователей (groups)** — объединение участников для массовой выдачи доступа
- **Автоматическая проверка решений** — код участника выполняется в изолированной среде (песочнице) на Python
- **Линтинг PEP8** — автоматическая проверка стиля кода через pycodestyle
- **Поддержка LaTeX** — математические формулы в условиях задач отображаются через KaTeX
- **Панель администратора** — полное управление пользователями, задачами, тестами, контестами
- **Личный кабинет участника** — просмотр задач, контестов, отправка решений, история попыток
- **SQLite** — не требуется отдельный сервер баз данных, всё хранится в одном файле

## Требования к окружению

### Обязательное ПО

- **Веб-сервер**: Apache 2.4+ с mod_rewrite, Nginx, или PHP CLI 8.0+
- **PHP**: версия 8.0 или выше с расширениями:
  - `pdo_sqlite` — для работы с SQLite
  - `mbstring` — для работы со строками
- **Python**: версия 3.6 или выше (для выполнения кода участников)
- **pycodestyle**: утилита для проверки стиля PEP8 (устанавливается через pip)

### Проверка зависимостей

```bash
# PHP и расширения
php -v
php -m | grep -i pdo
php -m | grep -i sqlite
php -m | grep -i mbstring

# Python
python3 --version

# pycodestyle
python3 -m pycodestyle --version
# или
pycodestyle --version
```

## Установка и настройка

### 1. Клонирование репозитория

```bash
git clone <repository-url> contest-web
cd contest-web
```

### 2. Настройка конфигурации

Отредактируйте файл `config.php` под ваше окружение:

```php
define('SITE_NAME', 'Контест');                    // Название сайта
define('BASE_URL', 'https://contest.example.com'); // Полный URL сайта (с протоколом)
define('DB_PATH', __DIR__ . '/data/contest.db');   // Путь к SQLite БД
define('SANDBOX_DIR', __DIR__ . '/sandbox');        // Директория для временных файлов

// Лимиты по умолчанию (можно переопределить для каждой задачи отдельно)
define('DEFAULT_TIME_LIMIT', 2.0);    // секунд
define('DEFAULT_MEMORY_LIMIT', 128);  // МБ

// Часовой пояс
define('DISPLAY_TIMEZONE_NAME', 'Europe/Samara');  // Часовой пояс для отображения
```

### 3. Права доступа

Убедитесь, что веб-сервер имеет право на запись в директории `data/` и `sandbox/`:

```bash
# Linux
chmod 775 data/ sandbox/
# или
chown -R www-data:www-data data/ sandbox/

# Windows (если используется XAMPP, OpenServer и т.п.)
# Обычно права уже настроены, проверьте что папки не Read-only
```

Директории создаются автоматически, но веб-сервер должен иметь права на их создание.

### 4. База данных

База данных SQLite (`data/contest.db`) создаётся автоматически при первом запуске. Таблицы инициализируются, и создаётся учётная запись администратора по умолчанию:

- **Логин**: `admin`
- **Пароль**: `admin`

> **Внимание:** В production-среде обязательно смените пароль администратора!

### 5. Запуск

#### Apache (рекомендуется)

Настройте VirtualHost так, чтобы корневой директорией была `contest-web/`:

```apache
<VirtualHost *:80>
    ServerName contest.example.com
    DocumentRoot "C:/path/to/contest-web"
    <Directory "C:/path/to/contest-web">
        Options Indexes FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>
</VirtualHost>
```

#### Встроенный PHP-сервер (для разработки)

```bash
php -S localhost:8080
```

Откройте в браузере: [http://localhost:8080](http://localhost:8080)

### 6. Установка pycodestyle (линтер)

Для проверки стиля кода (PEP8) необходим pycodestyle:

```bash
pip install pycodestyle
```

Или, если Python установлен глобально:

```bash
python -m pip install pycodestyle
# или
python3 -m pip install pycodestyle
```

**Как это работает**: при отправке решения сервер ищет pycodestyle в системе. Если линтер не найден, проверка стиля отключается. Рекомендуется установить его для всех пользователей системы.

## Развёртывание на хостинге

### Shared-хостинг (cPanel, ISPmanager и т.п.)

1. **Загрузите файлы** проекта через FTP/SFTP в корневую директорию сайта (обычно `public_html/`, `www/` или `htdocs/`)
2. **Проверьте PHP** — на большинстве хостингов PHP уже установлен. Убедитесь, что включено расширение `pdo_sqlite`:
   - В cPanel: раздел "Select PHP Version" → включите `pdo_sqlite`
   - В ISPmanager: раздел "PHP settings" → нужные модули
3. **Настройте Python**:
   - Узнайте путь к Python у поддержки хостинга
   - Также узнайте или установите pycodestyle (может потребоваться запрос в техподдержку)
   - Укажите путь к Python через переменную окружения `PYTHON_CMD` или отредактируйте метод `findPython()` в `includes/Sandbox.php`
4. **Настройте конфигурацию** — отредактируйте `config.php`:
   - `BASE_URL` → укажите полный URL вашего домена (например, `https://contest.example.com`)
   - При необходимости скорректируйте часовой пояс
5. **Права доступа** — установите права `755` на папки `data/` и `sandbox/` (или `775`, если PHP работает от другого пользователя)
6. **Убедитесь, что SQLite работает** — PHP должен иметь доступ на запись в директорию с БД

> **Важно:** На дешёвых shared-хостингах может не быть Python. В этом случае проверка решений работать не будет. Уточните у хостинг-провайдера возможность установки Python и запуска shell-команд из PHP (`exec`, `shell_exec`, `proc_open`).

### VPS / Выделенный сервер (Linux)

```bash
# 1. Установка веб-сервера и PHP
sudo apt update
sudo apt install -y apache2 php php-sqlite3 php-mbstring

# 2. Установка Python и линтера
sudo apt install -y python3 python3-pip
pip3 install pycodestyle

# 3. Размещение проекта
sudo mkdir -p /var/www/contest
sudo cp -r . /var/www/contest/
cd /var/www/contest

# 4. Права доступа
sudo chown -R www-data:www-data /var/www/contest
sudo chmod -R 755 /var/www/contest
sudo chmod 775 /var/www/contest/data
sudo chmod 775 /var/www/contest/sandbox

# 5. Настройка Apache
sudo nano /etc/apache2/sites-available/contest.conf
```

Пример конфигурации Apache:

```apache
<VirtualHost *:80>
    ServerName contest.example.com
    DocumentRoot /var/www/contest

    <Directory /var/www/contest>
        Options Indexes FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>

    ErrorLog ${APACHE_LOG_DIR}/contest-error.log
    CustomLog ${APACHE_LOG_DIR}/contest-access.log combined
</VirtualHost>
```

```bash
sudo a2ensite contest.conf
sudo a2enmod rewrite
sudo systemctl restart apache2
```

**Настройка Python на сервере:**

В файле `config.php` добавьте (или укажите через переменную окружения):

```php
putenv('PYTHON_CMD=/usr/bin/python3');
```

### Docker (альтернативный способ)

Если вы используете Docker, можно создать контейнер с PHP, Python и веб-сервером. Примерный `Dockerfile`:

```dockerfile
FROM php:8.2-apache

RUN docker-php-ext-install pdo_sqlite mbstring

RUN apt-get update && apt-get install -y \
    python3 \
    python3-pip \
    && pip3 install pycodestyle

COPY . /var/www/html/

RUN chown -R www-data:www-data /var/www/html/data /var/www/html/sandbox

EXPOSE 80
```

## Развёртывание в локальной сети

Для использования в локальной сети (например, в учебном классе) выполните шаги 1–4 из раздела [Установка и настройка](#установка-и-настройка). Затем:

### Способ 1: Apache + статический IP

1. Настройте статический IP на компьютере-сервере (например, `192.168.1.100`)
2. Настройте Apache (как описано выше)
3. Убедитесь, что порт 80 открыт в брандмауэре Windows
4. На компьютерах учеников откройте `http://192.168.1.100` в браузере

**Открытие порта 80 в брандмауэре Windows:**

```
Панель управления → Система и безопасность → Брандмауэр Защитника Windows
→ Дополнительные параметры → Правила для входящих подключений → Создать правило
→ Для порта → TCP → 80 → Разрешить подключение → Дать имя "HTTP Contest"
```

### Способ 2: Встроенный PHP-сервер (проще всего для локальной сети)

На компьютере-сервере выполните:

```bash
# Узнайте свой IP-адрес (например, 192.168.1.100)
ipconfig

# Запустите сервер на всех интерфейсах, порт 8080
php -S 0.0.0.0:8080
```

На компьютерах учеников откройте `http://192.168.1.100:8080`

### Настройка Python для локальной сети

Убедитесь, что Python установлен на сервере и доступен из командной строки:

```bash
# Проверить наличие Python
python --version
# или
python3 --version

# Установить линтер (рекомендуется)
pip install pycodestyle
```

## Использование

### Вход в систему

По умолчанию создаётся один администратор:

- **Логин**: `admin`
- **Пароль**: `admin`

### Администрирование

После входа с правами администратора вам доступны:

- **Управление пользователями** — создание, редактирование, удаление участников
- **Управление группами** — объединение пользователей в группы для массовой выдачи доступа
- **Управление задачами** — создание задач с описанием (с поддержкой LaTeX), тестами, лимитами
- **Управление группами задач** — логическая группировка задач по темам
- **Управление контестами** — создание контестов, привязка задач, настройка времени, выдача доступа
- **Просмотр решений** — все попытки участников с результатами тестов

### Для участников

После того как администратор создал контест и выдал доступ:

1. **Вход в систему** — используйте логин и пароль, выданные администратором
2. **Просмотр контестов** — на главной странице отображаются доступные контесты
3. **Решение задач** — выберите задачу, напишите код на Python, отправьте на проверку
4. **Результаты** — после проверки отображаются пройденные/проваленные тесты, статус, время выполнения
5. **История** — все отправленные решения сохраняются для последующего просмотра

## Безопасность

- **Песочница**: код участников выполняется в изолированном процессе Python с ограничением по времени и памяти. Запрещённые модули (`os`, `subprocess`, `sys`, `shutil`, `ctypes`, `signal`, `multiprocessing`, `threading`, `socket`) не блокируются на уровне ОС, а проверяются через линтер.
- **SQL-инъекции**: все запросы к БД выполняются через подготовленные выражения PDO.
- **XSS**: все выводимые пользовательские данные экранируются через `htmlspecialchars()`.
- **Сессии**: PHP-сессии с флагами `httponly` и `use_only_cookies`.
- **Валидация**: на стороне сервера проверяется существование задач, контестов и прав доступа.
- **Обновление sandbox**: при каждом запросе создаются уникальные временные файлы, которые удаляются после выполнения.

## Решение проблем

### База данных не создаётся

Убедитесь, что PHP имеет права на запись в директорию `data/`:

```bash
chmod 775 data/
```

### Python не найден

Укажите путь к Python в переменной окружения:

```php
// В config.php
putenv('PYTHON_CMD=C:/Python312/python.exe');  // Windows
putenv('PYTHON_CMD=/usr/bin/python3');          // Linux
```

### Линтинг не работает

Установите pycodestyle:

```bash
pip install pycodestyle
```

Проверьте, доступен ли он из командной строки:

```bash
pycodestyle --version
```

### Ошибка "Call to undefined function str_starts_with"

Обновите PHP до версии 8.0 или выше, либо замените `str_starts_with()` на `strpos()`.

### Отображается пустая страница

Включите отображение ошибок в `config.php`:

```php
ini_set('display_errors', 1);
ini_set('display_startup_errors', 1);
error_reporting(E_ALL);
```

### "500 Internal Server Error" при отправке решения

- Проверьте логи веб-сервера (в Apache: `/var/log/apache2/error.log`)
- Убедитесь, что Python установлен и доступен
- Проверьте права на запись в `sandbox/`
- Попробуйте запустить PHP-скрипт вручную, чтобы увидеть ошибки
